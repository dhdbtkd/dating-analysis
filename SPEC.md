# SPEC.md — 연애 유형 심층 분석 앱 설계서

---

## 개요

**서비스명**: 당신도 몰랐던 나의 연애 패턴
**형태**: 단일 HTML 파일 (output/index.html)
**목적**: ECR(Experience in Close Relationships) 척도 기반 성인 애착 유형 측정 + LLM 심층 대화로 "왜 나는 이런 연애를 하는가"를 깊이 탐색
**차별점**: 객관식 측정 후 LLM이 결과 컨텍스트를 가지고 사용자에게 질문을 던져 반성적 인사이트 생성. Claude 또는 OpenAI API 자유롭게 선택 가능.

---

## 학술 기반

- **측정 도구**: ECR-S (Experience in Close Relationships – Short Form)
  - 원출처: Brennan, K. A., Clark, C. L., & Shaver, P. R. (1998). Self-report measurement of adult attachment: An integrative overview. In J. A. Simpson & W. S. Rholes (Eds.), *Attachment theory and close relationships* (pp. 46–76). Guilford Press.
  - 단축형: Wei, M., Russell, D. W., Mallinckrodt, B., & Vogel, D. L. (2007). The Experiences in Close Relationship Scale (ECR)–short form: Reliability, validity, and factor structure. *Journal of Personality Assessment, 88*(2), 187–204.
- **유형 분류 체계**: Bartholomew, K., & Horowitz, L. M. (1991). Attachment styles among young adults: A test of a four-category model. *Journal of Personality and Social Psychology, 61*(2), 226–244.
- **2축 구조**:
  - **불안(Anxiety) 축**: 버림받을 것에 대한 두려움, 연인의 반응에 대한 과민한 모니터링
  - **회피(Avoidance) 축**: 친밀감에 대한 불편함, 감정 표현 억제, 자기 충족 강조

## LLM API 전환 기능 설계

### 인트로 화면 내 설정 영역
- LLM 제공자 선택 토글: **Claude** | **OpenAI**
- API 키 입력 필드 (선택된 제공자에 따라 플레이스홀더 변경)
  - Claude: `sk-ant-...`
  - OpenAI: `sk-...`
- API 키는 `sessionStorage`에만 저장 (페이지 닫으면 소멸)

### 엔드포인트 및 요청 포맷

**Claude (Anthropic)**
```
POST https://api.anthropic.com/v1/messages
Headers:
  x-api-key: {apiKey}
  anthropic-version: 2023-06-01
  content-type: application/json
  anthropic-dangerous-direct-browser-access: true
Body:
  model: claude-opus-4-5
  max_tokens: 1024
  system: {systemPrompt}
  messages: [{role, content}, ...]
```

**OpenAI**
```
POST https://api.openai.com/v1/chat/completions
Headers:
  Authorization: Bearer {apiKey}
  content-type: application/json
Body:
  model: gpt-4o
  max_tokens: 1024
  messages: [{role: "system", content: systemPrompt}, {role, content}, ...]
```

### 응답 파싱
- Claude: `response.content[0].text`
- OpenAI: `response.choices[0].message.content`
- 공통 파싱 함수 `extractText(response, provider)` 로 추상화

## 기능 목록

### 기능 1 — 인트로 화면
- 서비스명, 한 줄 설명
- 논문 출처 배지 2개 (Brennan et al. 1998 / Bartholomew & Horowitz 1991)
- 예상 소요시간 표시 ("약 7~10분")
- LLM 제공자 선택 + API 키 입력 UI
- "시작하기" CTA 버튼

### 기능 2 — 워밍업 문항 (3개)
- 워밍업 풀(W1~W10) 10개 중 3개 랜덤 추출
- 상황 기반 서술형 객관식 (4지선다)
- 점수 계산 없이 분위기 전환용 / 이후 LLM 컨텍스트 활용
- 진행 바 표시 (1/3, 2/3, 3/3)

### 기능 3 — ECR-S 측정 문항 (12개)
- 불안 풀(A1~A8)에서 6개, 회피 풀(V1~V8)에서 6개 랜덤 추출
- 7점 리커트 척도 (1=전혀 그렇지 않다 ~ 7=매우 그렇다)
- 역채점 문항(R 표시) 자동 처리: `score = 8 - rawScore`
- 진행 바 표시 (1/12 ~ 12/12)

### 기능 4 — 불안/회피 점수 계산 + 유형 판별
- 불안 점수 = 불안 문항 평균 (역채점 반영)
- 회피 점수 = 회피 문항 평균 (역채점 반영)
- 기준선 4.0 기준으로 4사분면 분류:
  - 불안 < 4 & 회피 < 4 → 🌿 따뜻한 안착형 (안정형)
  - 불안 ≥ 4 & 회피 < 4 → 🌪️ 소용돌이형 (집착형)
  - 불안 < 4 & 회피 ≥ 4 → 🏔️ 고독한 고지형 (거부-회피형)
  - 불안 ≥ 4 & 회피 ≥ 4 → 🎭 이중심리형 (두려움-회피형)

### 기능 5 — LLM 심층 대화 (6턴)
- ECR 측정 결과(불안 점수, 회피 점수, 유형, 워밍업 선택지)를 시스템 프롬프트에 주입
- 6턴 구조:
  - 턴 1~2: 최근 연애 경험 탐색 질문
  - 턴 3~4: 반복 패턴 / 회피 또는 집착 행동 탐색
  - 턴 5: 원가족 / 초기 애착 경험 탐색
  - 턴 6: 변화 가능성 / 이상적 연애 비전 질문
- 사용자 답변은 텍스트 입력창으로 받음
- 대화 기록 전체 누적하여 다음 턴 요청에 포함

### 기능 6 — 결과 JSON 생성 (LLM API 호출)
- 대화 완료 후 LLM에 JSON 형식으로 결과 생성 요청
- JSON 스키마:
```json
{
  "typeName": "감성 유형명",
  "tagline": "한 줄 저격 문장",
  "lovePattern": "연애 패턴 설명 (200자)",
  "coreWound": "핵심 상처/공포 (100자)",
  "actionTip": "구체적 행동 지침 3가지 (배열)",
  "mindset": "마인드셋 전환 문장 (100자)",
  "quote": "대화 중 인상적인 사용자 발언 직접 인용 또는 유형 대표 인용구",
  "famousMatch": "유명인/캐릭터 비교 (이름 + 한 줄)",
  "anxietyScore": 0.0,
  "avoidanceScore": 0.0
}
```
- JSON 파싱 실패 시 폴백: 정규식으로 `{...}` 추출 재시도, 그래도 실패 시 기본값 세트 사용

### 기능 7 — 결과 카드
- 유형 아이콘 + 감성 유형명 + 학술 유형명
- 태그라인 (저격 인용구)
- 2축 산포도 SVG:
  - X축: 회피, Y축: 불안 (각 1~7)
  - 4사분면 배경 컬러 (투명도 낮게)
  - 유명인/캐릭터 좌표 표시 (이름 레이블)
  - 사용자 좌표 강조 표시 (골드 원형 마커)
- 연애 패턴 / 핵심 상처 / 행동 팁 3가지 / 마인드셋
- LLM 대화 인용구 블록
- 유명인 비교

### 기능 8 — 바이럴 공유 기능
- "결과 공유하기" 버튼: Web Share API → 미지원 시 클립보드 복사
- 공유 텍스트 자동 생성: `[유형명] — 나의 연애 패턴 테스트 결과 | [URL]`
- "상대방도 해보기" 버튼: 링크 복사
- "커플 궁합 보기" 버튼: 두 유형 조합 설명 모달

### 기능 9 — LLM API 전환 (설정 UI)
- 인트로에서 Claude / OpenAI 선택 가능
- 선택에 따라 엔드포인트, 헤더, 파싱 로직 분기
- API 키 유효성 기본 검사 (형식 체크)
- 오류 발생 시 친절한 한국어 에러 메시지

### 불안(Anxiety) 풀 (A1~A8, 6개 랜덤 추출)

> 역채점 문항은 (R) 표시. 실제 점수 = 8 − 응답값

**A1.** 나는 연인에게 버림받을까 봐 걱정된다.
*(정채점)*

**A2.** 연인이 나를 사랑하지 않게 될까 봐 두렵다.
*(정채점)*

**A3.** 연인과 가까워지고 싶지만, 상대가 나만큼 원하지 않을까 봐 불안하다.
*(정채점)*

**A4.** 나는 연인에게 내가 중요한 사람인지 자주 확인받고 싶다.
*(정채점)*

**A5.** 연인이 나를 정말 소중히 여기는지 확신이 없다.
*(정채점)*

**A6.** 연인이 나를 떠날까 봐 때때로 걱정된다.
*(정채점)*

**A7 (R).** 나는 연인을 잃을까 봐 별로 걱정하지 않는다.
*(역채점 — 점수 = 8 − 응답값)*

**A8 (R).** 연인과 헤어질지도 모른다는 생각이 거의 들지 않는다.
*(역채점 — 점수 = 8 − 응답값)*

---

### 회피(Avoidance) 풀 (V1~V8, 6개 랜덤 추출)

> 역채점 문항은 (R) 표시.

**V1.** 나는 연인에게 내 감정을 드러내는 것이 불편하다.
*(정채점)*

**V2.** 나는 연인에게 완전히 의지하는 것이 어렵다.
*(정채점)*

**V3.** 나는 연인과 너무 가까워지는 것이 부담스럽다.
*(정채점)*

**V4.** 나는 연인에게 속마음을 털어놓기가 쉽지 않다.
*(정채점)*

**V5.** 친밀한 관계를 맺는 것이 편안하다.
*(역채점 — 점수 = 8 − 응답값)*

**V6.** 연인에게 의지하는 것이 편안하게 느껴진다.
*(역채점 — 점수 = 8 − 응답값)*

**V7.** 나는 연인과 가까워지는 것을 즐긴다.
*(역채점 — 점수 = 8 − 응답값)*

**V8.** 나는 연인이 나에게 의지하는 것이 불편하다.
*(정채점)*

## 기술 스택

- **단일 파일**: HTML + CSS + Vanilla JavaScript (외부 라이브러리 없음, CDN 폰트 제외)
- **상태 관리**: 전역 `appState` 객체
- **화면 전환**: `showScreen(id)` 함수로 단일 페이지 앱 구현
- **API 호출**: `fetch()` 비동기 (async/await)
- **결과 저장**: `sessionStorage` 활용
- **공유**: Web Share API + 클립보드 fallback
