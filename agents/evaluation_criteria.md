# 평가 기준표 — 연애 유형 심층 분석 앱

Generator와 Evaluator가 공유하는 품질 기준.

---

## 채점 항목

### 1. 디자인 품질 (비중: 40%)

합격:
- 어두운 배경 (#0a0a0f, #111118) + 골드 포인트 (#c8a96e) — 밤 감성
- Gowun Batang(제목) + Noto Sans KR(본문) 폰트 조합
- grain overlay (SVG noise, body::before 또는 ::after)
- orb 배경 효과 (radial-gradient blur, position: fixed)
- 결과 카드가 "저장하고 싶다"는 느낌 — 공유 욕구를 자극하는 비주얼
- 문항 선택 시 즉각적인 피드백 (선택 하이라이트 + 자동 다음 문항)
- 대화 버블이 실제 메신저처럼 느껴지는 UI

불합격:
- 보라색/파란색 그라데이션 배경
- 흰색 카드 격자 레이아웃
- Inter/Roboto 기본 폰트만 사용
- 밝은 배경 (연애 주제와 미스매치)
- 결과 카드가 평범한 텍스트 나열

### 2. 독창성 (비중: 30%)

합격:
- ECR 문항 풀 → 랜덤 추출 (매 세션 다른 문항 조합)
- 2축 산포도에 유명인/캐릭터 좌표가 연하게 찍혀 "나 vs 유명인" 비교
- Claude와의 실제 대화 내용이 결과 인용구로 등장 ("저격당한 느낌")
- 감성적 유형명 (학술명이 아닌 창작 이름: "소용돌이형", "고지형" 등)
- 결과 카드의 "대화에서 발견된 나" 섹션 — 이게 핵심 차별점
- 바이럴 공유 + 궁합 초대 구조

불합격:
- 단순 4분류 레이블만 표시 (인용구 없음)
- 유명인 없이 내 점수만 표시
- Claude API 연동 없이 하드코딩된 결과
- 학술 유형명 그대로 사용 (Secure, Preoccupied 등)
- 매 세션 동일한 문항 (풀 방식 미구현)

### 3. 기술적 완성도 (비중: 15%)

합격:
- 반응형 정상 작동 (모바일/태블릿/데스크톱)
- Claude API fetch 정상 호출 + 에러 핸들링
- 역채점 로직 점수 계산에 올바르게 반영
- JSON 파싱 실패 시 폴백 결과 표시
- 화면 전환 자연스러운 애니메이션 (fadeUp 0.5s)
- 대화 자동 스크롤 (chat-window scrollTop)
- 로딩 텍스트 순환 (setInterval)

불합격:
- 모바일에서 레이아웃 깨짐
- API 오류 시 흰 화면 또는 undefined 표시
- 역채점 미반영으로 점수 오류
- JSON.parse 에러 처리 없음

### 4. 기능성 (비중: 15%)

합격:
- 인트로 → 워밍업 → 측정 → 대화 → 결과 흐름 완주 가능
- 워밍업/측정 문항 선택 후 자동 또는 수동 다음 진행
- 대화 6턴 완료 시 자동으로 결과 화면으로 이동
- 결과 카드에 패턴/강점/주의/행동/마인드셋/인용구 모두 존재
- 논문 출처 하단 명시
- 공유 버튼 작동

불합격:
- 특정 단계에서 막힘 (다음 버튼 없음 등)
- 대화 끝나도 결과로 이동 안 됨
- 인용구 섹션 없음
- 논문 출처 없음

---

## 판정

가중 점수 = (디자인×0.4) + (독창성×0.3) + (기술×0.15) + (기능×0.15)

- 7.0 이상 → 합격
- 5.0~6.9 → 조건부 합격
- 5.0 미만 → 불합격
- 디자인 또는 독창성 4점 이하 → 무조건 불합격
- Claude API 연동 없음 → 무조건 불합격
- 대화 인용구 결과 카드 미표시 → 독창성 최대 5점
